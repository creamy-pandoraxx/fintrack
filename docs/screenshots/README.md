# Portfolio Screenshot Checklist

Replace this placeholder with real emulator or physical-device captures before publishing the final case study.

Recommended captures:

1. Welcome/login and registration validation.
2. Dashboard with representative summary cards, expense chart, and budget progress.
3. Wallet list and add/edit form.
4. Category tabs plus icon and color picker.
5. Transaction list, filters, and transaction form.
6. Budget list with normal and over-budget examples.
7. Realtime activity feed and finance tip.
8. Profile screen and delete-account confirmation.

Capture guidance:

- Use one consistent Android device frame and orientation.
- Prefer realistic demo data, not personal financial data.
- Redact email addresses, Firebase identifiers, tokens, LAN addresses, and credentials.
- Keep text large enough to read in a GitHub README.
- Include empty/loading/error states only when they support the engineering story.
- Use descriptive filenames such as `dashboard.png` and `category-picker.png`.
